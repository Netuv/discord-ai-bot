## 0. LATAR BELAKANG

### 0.1 Masalah

Komunitas Discord anime/manga/game/novel butuh konten segar, relevan, dan engaging secara konsisten — tanpa biaya operasional bulanan. Manual posting tidak scalable, sementara bot tradisional hanya repost RSS/feed tanpa kualitas.

### 0.2 Solusi

Autonomous content engine berbasis Cloudflare Workers yang:
- Menghasilkan artikel berkualitas via AI gratis (CF Workers AI, OpenRouter, NVIDIA NIM, Puter AI)
- Memilih topik + format secara cerdas berdasarkan history & trending signal
- Mendistribusikan ke Discord + social media via Composio
- Dikendalikan via MCP protocol (Claude Desktop, dll)
- 100% gratis beroperasi — hanya bayar domain jika perlu

### 0.3 Domain Masalah

| Aspek | Lingkup |
|-------|---------|
| **Domain** | Anime, manga, game, light novel — industri hiburan Jepang & global |
| **Target Pembaca** | Komunitas Discord Indonesia — santai, casual, engaging |
| **Frekuensi** | 1 artikel deep per 6 jam (cron) + manual trigger via MCP |
| **Bahasa Konten** | Bahasa Indonesia (casual, ngobrol) — kode: English |

### 0.4 Prinsip Desain

| Prinsip | Makna |
|---------|-------|
| **Quality > Quantity** | 1 artikel deep per trigger, bukan batch dangkal |
| **Budget-Aware** | Tiap subrequest di-track; hard stop di limit 50 |
| **Fail-Safe** | Semua fetch punya timeout; semua AI call punya fallback chain |
| **Zero-Cost Ops** | Hanya pakai free tier: CF Workers AI, OpenRouter, OpenCode, NVIDIA NIM, Puter AI |
| **Observable by Default** | TraceId per pipeline; D1 metrics per fase |
| **Extensible** | Plugin system untuk format & platform baru |

## 1. ARSITEKTUR APLIKASI

### 1.1 Arsitektur Global — Event-Driven Modular Monolith

Aplikasi ini menggunakan **Modular Monolith** dengan **Event-Driven pipeline** di dalam satu Cloudflare Worker.

**Kenapa Modular Monolith?**
- Satu deployment unit → lebih sederhana dari microservices
- Module dipisah per domain (`ai/`, `content/`, `composio/`, `mcp/`) → tetap terisolasi secara logika
- Komunikasi antar modul via function call langsung (bukan network)
- Cocok untuk resource-constrained environment (Worker CPU/memory terbatas)

**Pola Arsitektur yang Digunakan:**

| Pola | Penerapan |
|------|-----------|
| **Pipeline Pattern** | Content pipeline 5 fase berurutan: Strategist → Research → Generator → Publisher → Metrics |
| **Chain of Responsibility** | AI Model Router: coba provider satu per satu → fallback jika gagal |
| **Strategy Pattern** | Per-format ResearchEngine (12 implementasi), PlatformAdapter (5 implementasi) |
| **Adapter Pattern** | DiscordAdapter, twitter/instagram/linkedin/reddit/telegram adapters |
| **Plugin Architecture** | Format/Platform/Source plugins — registrasi via `plugin_registry` |
| **Circuit Breaker** | Provider auto-disable setelah 3 consecutive failures, aktif lagi setelah 5 menit |
| **Middleware Chain** | MCP router: Auth → Rate Limiter → Access Control → Audit Log |
| **Background Processing** | `ctx.waitUntil()` untuk non-blocking history + metrics logging |
| **Saga Pattern (mini)** | Pipeline punya fase terurut; jika satu fase gagal, seluruh pipeline gagal |
| **Repository Pattern** | D1Client abstraction over D1Database — query/execute/first/batch |
| **Cache-Aside** | D1Cache — baca dari cache dulu, fallback ke API, simpan ke cache |

### 1.2 System Layers

### 1.1 System Layers

```
TRIGGERS: Cron (0 */6 * * *) | MCP Tools | Webhooks | Discord Interactions | HTTP API
     │
MIDDLEWARE: CORS → Auth → Rate Limiter → Access Control → Audit Log
     │
ORCHESTRATOR: ContentStrategist → Research → Generator → Publisher → History/Metrics
     │
STORAGE: D1 (CONTENT_DB: 10 tables) + KV (BOT_KV: rate limits, audit, vision cache)
```

### 1.2 Content Pipeline Flow

```
T0  TRIGGER → T1 STRATEGIST (decide format/category/topic) → T2 RESEARCH (per-format engine + media)
→ T2.5 MEDIA SEARCH (YouTube + Images + AI Vision) → T3 GENERATOR (prompt→AI→parse→audit→fix)
→ T4 PUBLISH (Discord embed + sections) → T5 HISTORY + METRICS (ctx.waitUntil)
→ (optional) COMPOSIO DISTRIBUTION → Twitter/IG/LinkedIn/Reddit/Telegram
```

## 2. CODING STYLE & CONVENTIONS

### 2.1 Naming

| Element | Style | Example |
|---------|-------|---------|
| Files | kebab-case | `model-router.ts`, `topic-generator.ts` |
| Classes | PascalCase | `ContentStrategist`, `BudgetTracker`, `D1Client` |
| Functions/Methods | camelCase | `runArticlePipeline()`, `callAiWithRouter()` |
| Variables | camelCase | `traceId`, `channelId`, `brief` |
| Constants | UPPER_SNAKE | `MODEL_ROUTES`, `FORMAT_WEIGHTS`, `CORS_HEADERS` |
| Interfaces/Types | PascalCase | `ContentBrief`, `PipelineResult`, `ModelRoute` |
| Type aliases | PascalCase | `ContentCategory`, `PipelinePhase` |
| Enums | PascalCase enum + UPPER keys | `ErrorCode { BUDGET_EXHAUSTED = 'BUDGET_EXHAUSTED' }` |
| Env bindings | UPPER_SNAKE | `DISCORD_TOKEN`, `CONTENT_DB`, `AI` |
| DB columns | snake_case | `topic_normalized`, `published_at`, `run_count` |

### 2.2 TypeScript Config

- `strict: true`, `noUncheckedIndexedAccess: true`, `exactOptionalPropertyTypes: true`, `noImplicitReturns: true`
- Target: `ES2022`, Module: `ES2022`, ModuleResolution: `bundler`
- Types: `@cloudflare/workers-types` (no `@types/node`)
- **Pattern for optional fields:** `string | null` (not `?` optional) when DB nullable cols
- **Non-null assertion:** `arr[0]!` used after length check (e.g. `results[0]!`)
- **Async IIFEs:** Avoid; use top-level await or `ctx.waitUntil()` for fire-and-forget

## 5. MCP SERVER

### 5.1 Protocol
- JSON-RPC 2.0, protocol version `2024-11-05`
- Endpoint: `POST /mcp`
- Auth: `x-mcp-secret` header (compared to `env.MCP_SECRET`)
- Rate limit: 30 req/min per IP (KV-based)
- CORS: All origins, methods GET/POST/OPTIONS

### 5.2 Tool Permission Levels

| Level | Tools |
|-------|-------|
| **public** (no auth) | `status`, `health` |
| **user** (auth'd) | `ai-chat`, `generate-article`, `send-message`, `purge-channel`, `get-history`, `get-metrics`, `composio-status`, `composio-post`, `composio-broadcast` |
| **admin** | `task-list`, `task-create`, `task-toggle`, `task-delete`, `provider-health`, `clear-dlq`, `ban-user`, `kick-user`, `plugin-list`, `plugin-toggle` + everything below |

Default for unknown tools: `admin` (fail closed).

### 5.3 MCP Methods
- `initialize` → server info + capabilities
- `tools/list` → tools available to current role
- `tools/call` → execute tool by name

## 6. WRITING STYLE — CONTENT GENERATION

### 6.1 Tone Rules (System Prompt)
- **Language:** Indonesian casual — "ngobrol sama teman"
- **Voice:** Consistent "gue/lo" or "aku/kamu" (don't mix)
- **Hook:** Start with strong hook — surprising fact, sharp question, bold opinion
- **No-no openers:** "Halo", "Selamat datang", "Pada artikel ini", "Tentu saja"
- **No-no closers:** "Kesimpulannya", "Sekian", "Terima kasih", "Semoga bermanfaat"
- **Expression variety:** No repeating "Gue sih penasaran" or "jujur sih"; use varied expressions
- **Body:** 2-4 short paragraphs per section, natural flow, opinion + insight, not dry facts
- **Format:** Valid JSON output with `title`, `intro`, `sections[{heading, body, imageDescription?, videoQuery?}]`

### 6.2 Banned Phrases (Auditor)
`kesimpulannya, dapat disimpulkan, demikian artikel, sekian, terima kasih, that's all, semoga bermanfaat, generated by ai, as an ai, i am an ai, sebagai ai, pada artikel ini kita akan, halo semua`

### 6.3 Format-Specific Instructions (12 formats)
See `format-prompts.ts` — each format has distinct structure rules (e.g., `review` = hook → USP → analysis → debate → verdict; `breaking-news` = headine → impact → reactions → why now).

## 17. ANTI-PATTERNS TO AVOID

- ❌ `mcpServer.serve()` — does not exist in current MCP SDK; use `WebStandardStreamableHTTPServerTransport`
- ❌ Un-`await`ed `verifyKey()` — function is async; must `await`
- ❌ Throwing non-AppError without proper error code
- ❌ Configuring values that should be env vars (secrets, IDs, tokens)
- ❌ Making a subrequest without `budget.consume()`
- ❌ Using `?` optional fields when DB expects `string | null` (`exactOptionalPropertyTypes` breaks)
- ❌ Direct `console.log` without `traceLog()` (loses traceId context)
- ❌ Modifying global `MCPRequest.params.arguments` without cloning first
- ❌ Using `any` type; prefer `unknown` + type guard
